# user-service

Консольное Java-приложение для управления пользователями с использованием Hibernate и PostgreSQL без Spring.

Приложение реализует базовые CRUD-операции над сущностью `User` и предоставляет консольный интерфейс для взаимодействия с пользователем.

---

## Возможности

- Создание пользователя
- Получение списка всех пользователей
- Получение пользователя по ID
- Обновление данных пользователя
- Удаление пользователя
- Работа с PostgreSQL через Hibernate
- Настройка Hibernate без Spring
- Использование DAO-паттерна
- Транзакционная обработка операций
- Логирование
- Обработка ошибок Hibernate и PostgreSQL

## Запуск инфраструктуры через Docker Compose

Для локального запуска PostgreSQL и pgAdmin используется `docker-compose.yml`.

### Сервисы

- **PostgreSQL** — база данных приложения
- **pgAdmin** — веб-интерфейс для управления PostgreSQL

### Описание конфигурации

#### PostgreSQL
Контейнер PostgreSQL запускается на образе `postgres:16`.

Используются переменные окружения:

- `ASTON_POSTGRES_DB`
- `ASTON_POSTGRES_USER`
- `ASTON_POSTGRES_PASSWORD`

Порт базы данных проброшен как `5432:5432`.

Также подключаются:
- volume `postgres_data` для хранения данных
- `init.sql` для начальной инициализации базы

Healthcheck используется для проверки готовности базы к работе.

#### pgAdmin
Контейнер pgAdmin запускается на образе `dpage/pgadmin4:latest`.

Используются переменные окружения:

- `ASTON_PGADMIN_DEFAULT_EMAIL`
- `ASTON_PGADMIN_DEFAULT_PASSWORD`

Веб-интерфейс доступен на порту `5050`.

### Основные команды управления

- **Поднять всё окружение в фоновом режиме:**
  `docker compose up -d`

- **Проверить статус запущенных сервисов:**
  `docker compose ps`
 
- **Посмотреть логи (если что-то пошло не так):**
  `docker compose logs -f`

- **Остановить и удалить контейнеры, сохранив данные:**
  `docker compose down`

- **Остановить всё с удалением данных (полная очистка):**
  `docker compose down -v`
